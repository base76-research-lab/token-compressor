## Token Compressor — March 8, 2026 Update

What changed
- Robustness: deterministic rewrites now cover anchored/hallucination/math/transition edge cases; panel passes clean.
- Modes: new `process_prompt(mode=...)` with `raw | compressed | vector | compressed+vector`.
- Embeddings: `vectorize_prompt()` provides L2-normalized embeddings for pre-injection use.
- CLI: `cli.py --mode raw|compressed|vector|compressed+vector` (default `compressed`). Vector modes return JSON `{mode,text,vector}`.
- Docs/Schema: README and COMPRESSION_RESULT_SCHEMA updated to reflect thresholds (0.85) and new modes.
- MCP: `mcp` dependency included; server starts (`python3 -m token_compressor_mcp --help`).

How to pull and smoke-test
```bash
git pull origin main
python3 -m unittest discover -s tests -p 'test_*.py'
```

Quick usage
```bash
echo "Your long prompt" | python3 cli.py --mode compressed
echo "Your long prompt" | python3 cli.py --mode compressed+vector   # JSON with embedding
```

Notes
- Coverage floor stays at 0.85 overall; specific high-risk prompts use deterministic rewrites that bypass embedding validation.
- MCP tool does not yet expose `mode`; use CLI or Python API until added.
